## Context

当前香灰效果已经具备基础的随进度增长、三根香差异化和完成态掉落能力，但视觉结果仍主要呈现为贴近香杆的 S 形细曲线。结合参考图和现有效果，问题不在于燃烧状态机或烟雾粒子，而在于香灰几何模型仍把后半程演化表达成连续摆动，缺少短直挂灰、细颈、主环、次环和尾端下垂这些勾炉香的关键结构。

现有实现主要集中在 `cloud-incense/IncenseCanvasView.swift`，并由单一全局 `progress` 驱动。为了保持业务状态稳定，这次设计不引入独立的逐根物理仿真，而是在当前进度模型上重构香灰路径生成方式与绘制边界。

## Goals / Non-Goals

**Goals:**
- 将香灰的中后段视觉从 S 形摆动提升为可辨认的成圈挂灰。
- 保留早期短直挂灰，并在中期形成至少一个大环，后期出现次环或不完全闭合的尾段。
- 增强横向展开和重力下垂，使香灰在视觉上脱离香杆并表现为悬挂结构。
- 在不修改主流程状态的前提下，为三根香保留稳定但不完全相同的成圈差异。
- 保证完成态掉落延续最终成圈形态，而不是切换成另一套预设曲线。

**Non-Goals:**
- 不引入基于粒子、骨架或刚体约束的真实物理求解。
- 不修改点燃顺序、燃烧时长、烟雾倾斜响应或 Live Activity 行为。
- 不强制实现燃烧过程中的局部断裂掉灰，只为未来扩展预留结构空间。

## Decisions

### 1. 使用“环链式路径”替代连续波形中心线
- 决策：将香灰路径拆成直杆段、细颈过渡、主环、次环、尾端下垂几个显式阶段，而不是继续增强连续摆动曲线。
- 原因：连续波形和轻螺旋更容易读成 S 形。环链式路径能直接表达参考图中的大圈悬挂结构。
- 备选：继续调大现有 coil 半径和摆幅。未选，因为只能得到更夸张的 S 或螺旋，难以稳定生成近闭合大环。

### 2. 将香灰绘制边界从香杆宽度中解耦
- 决策：香灰视图的可绘制宽度必须明显大于香杆 frame，允许主环横向超出香杆本体。
- 原因：当前香灰受限在狭窄 frame 中时，只能产生轻微左右偏移，不足以形成大圈。
- 备选：维持当前 frame 宽度，仅靠路径计算在局部挤出。未选，因为渲染和布局边界会持续限制成圈幅度。

### 3. 保留单一 progress，但拆分为结构阶段参数
- 决策：继续使用全局 `progress`，并从中派生直杆强度、主环展开、次环形成、尾端下垂四类参数。
- 原因：当前状态模型已经满足业务流程需求。通过阶段参数化即可表达结构变化，而无需扩大状态面。
- 备选：为每根香单独维护形态状态机。未选，因为会增加耦合与重置复杂度，且超出这次变更范围。

### 4. 差异化从“小参数偏移”升级为“结构偏移”
- 决策：每根香的稳定 seed 不只控制半径和偏移，还要决定主环更圆或更扁、次环出现早晚、尾端更收还是更坠。
- 原因：只有结构差异，才能避免三根香看起来像同一路径的复制偏移。
- 备选：继续仅使用幅度与延迟差异。未选，因为仍容易显得模板化。

### 5. 完成态掉落承接最终环形路径
- 决策：complete 时继续沿用当前掉落时机，但掉落动画必须基于最终环形路径整体下移和淡出。
- 原因：如果完成时切换成另一套预设图形，会破坏前面积累出的结构连续性。
- 备选：燃尽后替换为独立掉落图层。未选，因为视觉跳变明显。

## Risks / Trade-offs

- [环链路径参数过多导致调参困难] → 限制核心参数数量，只保留直杆长度、主环半径、次环缩放、尾端下垂和 seed 结构偏移。
- [香灰横向展开后与三根香之间发生视觉碰撞] → 约束主环偏向各自外侧，并限制最大展开范围不穿入相邻香杆。
- [更大的路径范围引起布局或裁切问题] → 在设计中明确放宽香灰绘制 frame，并在回归中校验不同进度下无裁切。
- [缺少真实断裂仍可能被感知为不够自然] → 本次先解决“从 S 到成圈”的主要辨识度问题，断裂掉灰留作后续可选增强。

## Migration Plan

- 在变更目录中先补充 `ash-evolution` 的规格要求，确保验收标准明确。
- 实现时仅替换香灰路径和其绘制边界，不修改 `BurnSession` 或烟雾系统接口。
- 如新几何模型出现明显回归，可临时退回旧路径生成逻辑，不影响主流程状态。

## Open Questions

- 主环是否必须在所有中后段都接近闭合，还是允许部分香灰呈现更松散的开环结构。
- 次环的优先级是否应高于尾端下垂，取决于最终预览中哪种更接近参考图。
- 是否需要在后续单独立项，为完成前最后数秒增加轻微失稳下沉动画。
