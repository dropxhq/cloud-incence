## Context

现有实现中，烟雾联动由 `TiltManager` 提供 roll/pitch，再由 `SmokeScene` 将其映射为粒子横向加速度与发射角。该路径在设备姿态与屏幕坐标一致时表现可接受，但在横屏（尤其左横屏）下会出现轴向错位，导致视觉上出现“烟向下飘”。

Live Activity 目前以阶段 + 倒计时文本为主，信息结构偏时间读数，不利于灵动岛快速扫视；同时未承载祈愿内容，过程情绪表达不连续。

该变更跨越 CoreMotion（输入）、SpriteKit（渲染）与 ActivityKit（系统展示）三个模块，属于跨层联动优化。

## Goals / Non-Goals

**Goals:**
- 在任意设备方向下保证烟雾主趋势向上。
- 让倾斜仅影响横向漂移与扰动强度，不再导致主方向下坠。
- 将灵动岛展示从倒计时文本切换为进度条表达。
- 支持在灵动岛/锁屏展示“正在祈祷”的简短内容（截断后）。
- 维持不支持能力设备的降级路径稳定可用。

**Non-Goals:**
- 不改变 21 分钟燃烧时长与点香主流程。
- 不引入 Live Activity 交互按钮或远程推送更新。
- 不实现全文祈愿展示与跨设备同步。

## Decisions

### 1) 使用重力向量进行“设备坐标 -> 屏幕坐标”映射
- 选择：优先使用重力向量（或由姿态推导等效向量）并按 portrait / landscapeLeft / landscapeRight 映射到屏幕 X/Y。
- 原因：roll/pitch 在横屏下语义漂移明显，向量映射更稳定、可解释。
- 备选：继续直接使用 roll/pitch 并做符号修补。未选，边界多且可维护性差。

### 2) 烟雾受力改为“向上主导 + 横向扰动”
- 选择：固定上升分量下限，横向偏移只调 xAcceleration；发射角只做小范围偏移或保持常量。
- 原因：满足“烟始终向上”的核心体验，同时保留倾斜反馈。
- 备选：允许发射角大范围随姿态摆动。未选，会在横屏和大角度时产生下坠错觉。

### 3) 灵动岛改用进度条语义，移除倒计时文本
- 选择：在 Expanded/Lock Screen 区域展示进度条和阶段文案，不再显示秒级倒计时文本。
- 原因：进度条更适合 glanceable UI，减少认知负担。
- 备选：进度条 + 倒计时并存。未选，布局拥挤且与本次目标冲突。

### 4) 扩展 Live Activity 状态字段承载祈愿摘要
- 选择：新增祈愿摘要字段（短文本，长度限制与截断）用于灵动岛/锁屏展示。
- 原因：将“祈愿”从输入态延续到燃烧态，增强仪式连续性。
- 备选：仅显示固定文案。未选，个性化不足。

## Risks / Trade-offs

- [方向映射在特定设备方向仍异常] → 建立方向映射表并覆盖 portrait/landscape 两侧场景测试。
- [进度条在紧凑岛区域可读性不足] → 紧凑区降级为极简符号，仅在 expanded/锁屏展示完整进度条。
- [祈愿文本过长造成截断语义损失] → 采用固定最大长度 + 尾部省略，必要时支持隐私开关。
- [更新频率过高影响电量] → 限制 Activity 更新节奏，进度视觉由系统时间区间推导优先。

## Migration Plan

1. 在变更中先完成 spec 对齐（方向映射、上升约束、进度条展示、祈愿摘要字段）。
2. 按模块分步落地：Tilt 输入归一化 -> Smoke 受力重构 -> Live Activity 数据模型扩展 -> Widget 展示替换。
3. 回归验证通过后上线；若出现兼容问题，可临时回退为“固定上升 + 无倾斜映射”与“阶段文案简版”。

## Open Questions

- 祈愿摘要默认是否开启显示，是否需要用户级隐私开关。
- 进度条样式采用线性条还是分段状态条（起香/正燃/将尽）。
- 是否需要在完成态短暂展示“祈愿已上达”后自动收起。
